## Centroid method note

Centroids are not polygon-geometric centroids at v0.3.0. The mapthai package stores polygons in R `sf` simple-features objects which the Python `pyreadr` library cannot parse. As the documented OSS-only fallback, centroids are computed as the median of subdistrict coordinates per province from `kongvut/thai-province-data` (7,124 of 7,452 subdistricts have coordinates). Bangkok has no subdistrict coordinates in kongvut and uses Wikidata Q1861 P625 (13.75, 100.5167) as a documented one-off. Polygon-geometric centroids will replace these in v1.0.0 once an R bridge or an alternative MIT-licensed polygon source is wired.
